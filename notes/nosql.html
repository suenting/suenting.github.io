NoSQL

NoSQL is a database which doesn't use relational tabels as the mechanism for storing data. The reason for this is to simplify the design as well as scaling across multiple machines for better performance. There are various types on NoSQL databases including Key-value stores, and Key-document stores. Many NoSQL databases (Hadoop, MongoDB) are compatible with a technique known as Map Reduce which is used to process large volumes of aggregated data which may not necissarily be on a single machine.

Map Reduce

Map Reduce is a high level programming model and implementation for large scale parallel data processing. The Map Reduce technique is broken up into two parts. The Mapper which is responsible for mapping data to an intermediate key value (this is essentially to divide the work into segments, somewhat similar to per-pixel operations of a pixel shader). The Reducer which performs an aggregated operation on all data which was mapped to its intermediate key, producing an output for said intermediate value.

Map( inputKey, inputValue):
	// - calculate intermediate value based on function of inputKey and inputValue
	EmitIntermediate( intermediateKey, relevantValue)

	// - perform aggregate operation on input collection
	Emit(outputKey, result)

Dynamo

As a case example, Dynamo is a popular NoSQL database built by Amazon based on a key-value data model, and offers high availability at the expence of consistency. Dynamo utilizes the methodology of having data eventually consistent, that is to say data across all servers will eventually be sync'd but this isn't required to happen right away all the time. For example, Dynamo allows the amazon store to be achive high availability when a user is browsing their store, and adding items to their cart. However their data in the cart is not required to be updated across all servers immediately, untill the user is ready to checkout. Therefore as operations do not need to block the systems progress untill checkout this system can achive better availability to provide a better user experience.

MongoDB

As a case example, MongoDB is a popular NoSQL database based on a key-document (BSON = Binary JSON) model. MongoDB is divided into three core components: mongod the database process, mongos the sharding controller, mongo the database shell (uses interactive javascript for administative tasks). For applications MongoDB uses object oriented drivers to act as an API and interact with the database.

Additional notes:
MongoDB uses memory mapped files, that is to say the total data accessible must be mapped to a virtual memory address (32-bit build only able to access 2GB).
MongoDB is not ACID, Atomic operations are only possible within the scope of a single document.
MongoDB automatically utilizes all free memory on a system.
